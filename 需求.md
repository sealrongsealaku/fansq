# 互联网支教教学反思墙系统需求文档（一期精简版）

## 一、项目名称

互联网支教教学反思墙系统

## 二、项目目标

建设一个面向互联网支教场景的教学反思墙系统，依托 QQ 群机器人完成学生反思内容采集，并通过后台人工审核后在前台页面展示，形成“提交、存储、审核、展示”的闭环。

## 三、一期定位

一期只聚焦核心链路：

**学生在 QQ 群中 @机器人提交反思 -> 系统自动接收并入库 -> 管理员后台人工审核 -> 前台反思墙展示**

一期不建设评论、排行、自动分析、音视频上传、多平台联动等功能。

## 四、用户角色

### 1. 学生用户

在指定 QQ 群中通过 `@机器人 + 反思内容` 的方式提交教学反思。

### 2. 管理员用户

登录后台，对提交内容进行查看、审核、编辑、发布和管理。

### 3. 浏览用户

浏览前台反思墙页面，查看审核通过后的展示内容，并可对展示内容点赞。

## 五、业务流程

1. 学生在指定 QQ 群中 `@机器人` 并发送反思内容。
2. 机器人服务接收群消息，解析反思正文和提交人信息。
3. 系统将反思内容写入数据库，状态默认为“待审核”。
4. 管理员在后台查看待审核内容并进行人工审核。
5. 审核通过且允许展示的内容在前台反思墙页面展示。
6. 浏览用户可对已展示内容进行点赞，且同一用户不可重复点赞。

## 六、提交方式需求

一期只采用一种提交方式：**QQ 群 @机器人提交**。

提交要求如下：

1. 学生无需跳转网页或填写额外表单。
2. 系统从 QQ 群消息中提取反思正文。
3. 提交人信息以学生在群中的 **QQ 昵称** 为准。
4. 支持同一学生多次提交反思。
5. 机器人可在提交成功后自动回复提示信息，如“已收到反思”。

## 七、QQ 机器人接入需求

### 1. 接入方式

一期采用 **WebSocket 模式** 接入 QQ 平台，由部署在校内服务器上的机器人服务主动建立长连接。

### 2. 接入要求

1. 机器人服务启动后主动连接 QQ 平台网关。
2. 鉴权成功后保持在线。
3. 收到群内 `@机器人` 消息后解析正文。
4. 将解析结果写入数据库，状态默认为“待审核”。
5. 支持异常断线后的自动重连。

## 八、功能需求

### 8.1 反思提交功能

1. 支持学生通过 QQ 群 `@机器人` 提交反思。
2. 支持记录提交时间、提交来源和提交内容。
3. 支持记录提交人在群中的 QQ 昵称。
4. 支持基础内容校验，例如空内容校验。
5. 提交成功后可返回提示信息。

### 8.2 内容存储功能

系统应将学生提交内容自动入库，至少保存以下字段：

- id
- submit_content
- submit_time
- submit_channel
- student_name
- source_group_id
- source_group_name
- raw_message_id
- audit_status
- display_status
- is_featured
- is_top
- is_anonymous
- display_name
- like_count
- remarks
- created_at
- updated_at

说明：

1. `student_name` 表示学生在 QQ 群中的昵称。
2. 所有内容应可追溯、可查询、可管理。

### 8.3 后台审核管理功能

1. 所有新提交内容默认进入“待审核”状态。
2. 管理员可执行以下操作：
   - 审核通过
   - 驳回或隐藏
   - 编辑修改
   - 删除
   - 设置精选
   - 设置置顶
   - 设置匿名展示名称
3. 支持按时间、群聊来源、审核状态、展示状态进行筛选。
4. 支持关键词搜索。
5. 支持查看单条内容详情。
6. 内容审核采用 **管理员人工审核**，一期不做敏感词检测。
7. 未审核内容不得直接进入前台展示。

### 8.4 前台反思墙展示功能

1. 前台只展示审核通过且允许展示的内容。
2. 页面整体风格应温暖、简洁，突出教育气质和成长记录感。
3. 支持卡片式展示。
4. 支持按时间排序。
5. 支持精选内容突出展示。
6. 支持显示 QQ 昵称或匿名名称。
7. 支持电脑端、移动端和大屏展示。
8. 支持自动滚动、轮播或分页等基础展示方式。
9. 支持展示项目名称、主题说明、时间范围、主办单位等辅助信息。

### 8.5 点赞功能

1. 前台已展示的反思内容支持点赞。
2. 点赞功能仅面向公开展示内容开放。
3. 系统应限制同一用户对同一条反思重复点赞。
4. 前台可展示每条反思的点赞数量。
5. 点赞功能为轻互动能力，一期不扩展评论和排行榜。

## 九、页面设计要求

前台页面定位为一个具有温度、阅读感和成长记录感的数字化教学反思展示空间。

建议页面结构如下：

1. 顶部引导区：展示项目名称、标题、说明文字。
2. 中部内容区：以卡片墙形式展示审核通过的反思内容。
3. 底部说明区：展示项目说明、时间范围、主办单位、二维码等信息。

设计关键词：

**温暖、安静、成长、陪伴、记录感、教育气质**

## 十、统计需求

一期只保留基础统计能力，支持统计以下内容：

1. 累计提交数量
2. 指定 QQ 群提交数量
3. 每日 / 每周提交趋势
4. 精选反思数量
5. 点赞总数量

## 十一、非功能需求

### 1. 安全性

1. 系统应具备管理员登录认证机制。
2. 未审核内容不得公开展示。
3. 后台应限制未授权用户访问。
4. 系统应具备基础日志记录能力。
5. 系统应具备数据库备份能力。

### 2. 稳定性

1. 反思提交后应稳定入库。
2. 后台审核操作应保存准确。
3. 前台页面应可稳定访问。
4. 机器人服务应支持掉线自动重连。

### 3. 可维护性

1. 系统结构应清晰，便于后续维护。
2. 页面展示主题和审核规则应支持调整。
3. 服务应支持日志查看和故障排查。
4. 部署方式应适配学校现有 Ubuntu 服务器环境。

### 4. 可扩展性

后续可按需要扩展：

1. 更多展示主题
2. 图片等多媒体内容
3. 教师点评
4. 标签分类
5. 智能分析

## 十二、部署需求

### 1. 部署环境

一期部署于学校现有 Ubuntu 服务器环境。

### 2. 推荐架构

1. Nginx 作为统一访问入口
2. 前端页面静态化部署
3. 后端服务提供业务逻辑与接口
4. MySQL 或 PostgreSQL 存储数据
5. systemd 负责服务守护
6. QQ 机器人服务独立运行，并通过 WebSocket 接入 QQ 平台

## 十三、一期建设重点

一期重点不是做复杂功能，而是稳定打通以下闭环：

**QQ 群 @机器人提交 -> 系统接收 -> 数据入库 -> 后台人工审核 -> 前台展示**

## 十四、一句话概括

建设一个基于 QQ 群 `@机器人` 提交的教学反思墙系统，以 QQ 昵称作为提交人信息，通过后台人工审核后进行前台展示，并支持不可重复的前台点赞功能。
